# Create tasks in a mentorship relation and complete the task workflow

Steps

1. Registration/SignUp in app
2. LogIn in app
3. Navigating to members section
4. Sending request to mentor/mentee
5. Viewing request
6. Accepting/Rejecting request
7. Creating task
8. Task completion

## Step 1 - Registration/SignUp in app

First step is to Register/SignUp in the app(if not already registered), in order to access the awesome features of the app

<img src="https://user-images.githubusercontent.com/37622734/93125847-4e518680-f6e9-11ea-9a3b-1bf0dc74da69.png" alt="SignUp Screen" width="200"/>

## Step 2 - Login in app

Once you have verified your email address, you can login into the app and access all the features

<img src="https://user-images.githubusercontent.com/37622734/93125812-4abdff80-f6e9-11ea-9f64-86f5beb6ad8e.png" alt="LogIn Screen" width="200"/>

## Step 3 - Navigating to members section

After logging into the app, navigate to the Members section to find members you would potentially want to connect with as mentor/mentee

<img src="https://user-images.githubusercontent.com/37622734/93125825-4bef2c80-f6e9-11ea-95f2-034986147e9d.png" alt="Members Section" width="200"/>

You can check member's profile by clicking on member's name

<img src="https://user-images.githubusercontent.com/37622734/93125822-4b569600-f6e9-11ea-89fc-af94d3f52750.png" alt="Member Profile" width="200"/>

## Step 4 - Sending request to mentor/mentee

Once you find out the member you want to connect to, open their profile and click `Send Request` button

<img src="https://user-images.githubusercontent.com/37622734/93125843-4db8f000-f6e9-11ea-8e6a-8312c2e3d36d.png" alt="Send Request" width="200"/>

Once request is sent, you can check the status of your request in Mentorship Requests screen

<img src="https://user-images.githubusercontent.com/37622734/93125840-4d205980-f6e9-11ea-809d-73c733d8f557.png" alt="Mentorship Requests Screen" width="200"/>

Also, you can view count of all requests in Home Screen

<img src="https://user-images.githubusercontent.com/37622734/93125805-4a256900-f6e9-11ea-91cb-3577b0eb44a9.png" alt="Home Screen" width="200"/>

Receiver of request will see following screen

<img src="https://user-images.githubusercontent.com/37622734/93125835-4c87c300-f6e9-11ea-8b57-d77027dd9106.png" alt="Receiver of Mentorship Requests" width="200"/>

## Step 5 - Viewing request

The receiver of request will be able to view the request details by clicking on it from Mentorsip Requests screen

<img src="https://user-images.githubusercontent.com/37622734/93125857-4f82b380-f6e9-11ea-8d5b-a8d2e9d5b9f4.png" alt="View Request" width="200"/>

## Step 6 - Accepting/Rejecting request

The receiver can either Accept or Reject the request. Once a request is accepted a new Relation is created

<img src="https://user-images.githubusercontent.com/37622734/93125831-4c87c300-f6e9-11ea-88db-47e17bd63c28.png" alt="Accepted Request - Relation" width="200"/>

## Step 7 - Creating Task

Now, the mentor can create tasks for the mentee from the Tasks Screen

<img src="https://user-images.githubusercontent.com/37622734/93125855-4eea1d00-f6e9-11ea-9220-83308f837ddb.png" alt="Task Screen" width="200"/>

Click on `+` icon to create task

<img src="https://user-images.githubusercontent.com/37622734/93125801-48f43c00-f6e9-11ea-937a-68e05e6c4177.png" alt="Create Task" width="200"/>

Once the task is created it will be shown to Mentee in To do list under Tasks tab of Current Relation

<img src="https://user-images.githubusercontent.com/37622734/93125851-4e518680-f6e9-11ea-825a-b8fc6d36fbd2.png" alt="Task Screen with tasks" width="200"/>

## Step 8 - Task completion

After completing the task, mentee can mark it as completed

<img src="https://user-images.githubusercontent.com/37622734/93125817-4abdff80-f6e9-11ea-858c-3a582a14357a.png" alt="Mark Task as Completed" width="200"/>

and the task will be moved from To do list to Achievements

<img src="https://user-images.githubusercontent.com/37622734/93125793-4560b500-f6e9-11ea-8e80-825f182a0601.png" alt="Completed Task" width="200"/>
